## **PRD：游戏开发者个人作品网站**

**版本：** 1.0
**日期：** 2024.07.10
**作者：** Spy

------

### 1. 项目概述

- **目标**：创建一个专业、视觉突出的个人作品网站，用于展示游戏开发项目、技术能力与职业身份，吸引潜在雇主、合作者或玩家。
- **定位**：简洁现代、交互流畅、突出游戏项目可视化的开发者作品集。
- **核心差异点**：强调游戏项目的可交互预览、视频展示、技术分解（如引擎、编程语言、美术工具）。

------

### 2. 用户角色与场景

| 角色        | 需求场景                               |
| :---------- | :------------------------------------- |
| 招聘方/HR   | 快速查看项目经验、技术栈、作品质量     |
| 同行/合作者 | 了解专业能力、项目细节、开发流程       |
| 玩家/爱好者 | 试玩游戏Demo、观看预告片、了解项目故事 |

------

### 3. 网站结构与页面规划

#### 3.1 全局元素

- **导航栏**：固定顶部，包含：首页(Logo)、项目、关于、联系
- **页脚**：版权信息、社交媒体图标（GitHub、LinkedIn、Twitter/Bluesky、游戏平台主页）

#### 3.2 页面内容

- **首页(Hero区)**：
  - 大标题 + 简短身份描述（如“游戏开发者，专注于 Unity 与 Unreal 引擎”）
  - 动态背景（可选：低多边形游戏场景循环动画）
  - “查看项目”按钮
- **项目展示页**：
  - 筛选标签（如：所有、Unity、Unreal、移动游戏、PC/主机）
  - 项目卡片：每个项目包含：
    - 项目名称 + 简短描述
    - **关键特色**：游戏类型、引擎、担任角色（程序/策划/美术等）
    - 封面图/动态GIF/视频悬停预览
    - 链接：Demo试玩、视频预告、详情页
- **项目详情页（点击项目后弹出或跳转）**：
  - 项目视频/可嵌入的WebGL试玩
  - 项目故事：开发动机、核心玩法、技术挑战与解决方案
  - 技术栈图标展示
  - 团队贡献说明（如果是团队项目）
- **关于页**：
  - 个人介绍 + 照片
  - 技术能力标签云（如C#、C++、Unity、Unreal、Shader编程等）
  - 职业经历时间线（可选）
- **联系页**：
  - 联系方式：邮箱、社交媒体
  - 联系表单（可选，防垃圾）
  - 当前状态（如“开放全职机会”或“可接 freelance 游戏原型开发”）

------

### 4. 功能需求清单

#### 高优先级

1. 响应式设计（桌面/平板/手机）
2. 项目筛选与分类功能
3. 视频/WebGL嵌入支持
4. 平滑滚动与页面过渡动画
5. SEO基础优化（标题、描述、结构化数据）

#### 中优先级

1. 暗色/亮色主题切换
2. 项目详情页的弹窗式设计（无需跳转）
3. 访客数据统计（如Google Analytics）

#### 低优先级

1. 博客系统（技术分享）
2. 多语言支持
3. 3D交互元素（如Three.js背景）

------

### 5. 视觉与交互设计参考

- **风格基调**：科技感、游戏化元素、干净布局
- **颜色方案**：深色背景（#0a192f 类似参考站） + 高亮色（如游戏UI常见的霓虹蓝/绿/紫）
- **字体**：无衬线字体（如Inter, Roboto Mono 用于代码）
- **动画**：悬停效果、滚动视差、渐进加载
